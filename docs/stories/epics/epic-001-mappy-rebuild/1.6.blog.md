## Story

**As a** visitante buscando informações sobre teleatendimento,
**I want** acessar artigos de blog informativos bem estruturados,
**so that** a Mappy ganhe autoridade no Google e em IAs (GEO).

## Acceptance Criteria

1. Rota `/blog` com listagem de posts (cards com imagem, título, data, resumo)
2. Rota `/blog/[slug]` com post individual renderizado via MDX
3. Posts escritos em Markdown com frontmatter (título, data, resumo, imagem, autor)
4. Meta tags SEO em cada post (title, description, og:image, canonical)
5. Schema markup `Article` (JSON-LD) em cada post
6. Listagem com paginação ou "carregar mais" (se >6 posts)
7. Design dos cards: foto + título + data + resumo — estilo Memorial Mais (clean)
8. Mobile responsivo
9. `npm run build` passa sem erros (usa next-mdx-remote)

## Dev Notes

**Contentlayer config básico:**
```ts
// contentlayer.config.ts
import { defineDocumentType, makeSource } from 'contentlayer/source-files'

export const Post = defineDocumentType(() => ({
  name: 'Post',
  filePathPattern: 'blog/*.mdx',
  contentType: 'mdx',
  fields: {
    title: { type: 'string', required: true },
    date: { type: 'date', required: true },
    excerpt: { type: 'string', required: true },
    image: { type: 'string' },
    author: { type: 'string', default: 'Equipe Mappy' },
  },
  computedFields: {
    slug: { type: 'string', resolve: (doc) => doc._raw.flattenedPath.replace('blog/', '') },
    url: { type: 'string', resolve: (doc) => `/blog/${doc._raw.flattenedPath.replace('blog/', '')}` },
  },
}))

export default makeSource({ contentDirPath: 'content', documentTypes: [Post] })
```

**SEO meta tags (Next.js 14):**
```tsx
export async function generateMetadata({ params }): Promise<Metadata> {
  const post = allPosts.find(p => p.slug === params.slug)
  return {
    title: `${post.title} | Mappy`,
    description: post.excerpt,
    openGraph: { images: [post.image] }
  }
}
```

**Foco de SEO (referência: rand-fishkin + kevin-indig):**
- Long-tail keywords: "telemedicina para empresas", "teleatendimento white-label", "benefício de saúde digital"
- Estrutura de heading hierárquica: H1 → H2 → H3
- Schema markup Article com datePublished, author, publisher
