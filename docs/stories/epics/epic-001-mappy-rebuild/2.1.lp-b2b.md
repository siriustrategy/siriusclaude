## Story

**As a** decisor de uma empresa (CEO, RH, Telecom, Clube de Benefícios),
**I want** ver uma landing page focada em como a Mappy serve minha empresa,
**so that** eu entre em contato rapidamente e solicite uma proposta.

## Acceptance Criteria

1. Rota `/empresas` (ou `/b2b`)
2. Ticker/marquee no topo: "Implementação em 5 dias · White-label 100% com sua marca · NPS 92,4% · +50k atendimentos"
3. Nav minimalista: logo + "Já tenho conta" + CTA "Solicitar Proposta"
4. Hero: headline com palavra colorida + imagem B2B real + form lateral
5. Headline: "Teleatendimento completo com **a sua marca**" (palavra colorida = "a sua marca" em #35B0FF)
6. Seção #35B0FF com features incluso + métricas
7. Componente "Ideal Para" reutilizado (Story 1.4)
8. Seção de depoimentos/cases (pode ser placeholder inicialmente)
9. FAQ com objeções B2B (accordion)
10. CTA final → WhatsApp + form B2B
11. Form B2B salva em Supabase `b2b_leads`
12. Sem footer padrão do site — footer minimalista da LP

**Headline com palavra colorida:**

**FAQ B2B — objeções a responder:**
1. "Preciso de CNPJ para contratar?" → Sim, a Mappy atende empresas
2. "Quanto tempo leva para implementar?" → 5 dias úteis após assinatura
3. "Posso usar minha própria marca?" → Sim, 100% white-label
4. "Qual o mínimo de vidas?" → A partir de 50 vidas
5. "Tem integração com sistemas de RH?" → Sim, via API
6. "E se meu cliente precisar atendimento às 3h da manhã?" → Medicina 24h disponível
7. "Como funciona o suporte?" → Suporte técnico dedicado durante implementação

**Form B2B → WhatsApp ao enviar:**
```ts
const buildB2BWA = (data: B2BLeadData) =>
  `https://wa.me/552231993772?text=${encodeURIComponent(
    `*Lead B2B*\nNome: ${data.nome}\nEmpresa: ${data.empresa}\nCargo: ${data.cargo}\nSegmento: ${data.segmento}\nWhatsApp: ${data.whatsapp}\nVidas: ${data.quantidade_vidas}`
  )}`
```
