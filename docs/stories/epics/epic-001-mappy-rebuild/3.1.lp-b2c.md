## Story

**As a** pessoa física buscando acesso a teleatendimento,
**I want** ver uma landing page clara com preços e benefícios,
**so that** eu assine o plano diretamente online.

## Acceptance Criteria

1. Rota `/assinar` (ou `/planos`)
2. Nav minimalista: logo + "Já sou cliente" + "Assinar Agora" (scroll to planos)
3. Hero: badge "100% Online" + headline + preço em destaque + imagem/video
4. Preço visível no hero: "por apenas **R$139,90/mês**" (preço em cor destaque)
5. Seção "O que está incluso": checklist visual dos serviços
6. Seção "Como funciona": 3 passos (Assina → Baixa app → Agenda)
7. Seção comparação de preço: "menos que R$4,67/dia" vs consulta avulsa
8. Cards de planos: Cartão recorrente R$139,90 + PIX/boleto R$149,90
9. FAQ com objeções B2C (accordion) — mínimo 7 perguntas
10. CTA "Assinar Agora" conectado ao Asaas (Story 3.2)
11. Design referência: saudemental.cuidarhoje.com.br


**Planos:**

**FAQ B2C — objeções a responder:**
1. "Posso cancelar quando quiser?" → Sim, sem fidelidade ou multa
2. "Quanto tempo demora para ativar?" → Acesso imediato após pagamento
3. "Funciona para minha família também?" → Consulte os planos família
4. "Precisa de plano de saúde?" → Não, funciona independentemente
5. "Como funciona a consulta online?" → Via app, em qualquer dispositivo
6. "Quais especialidades estão disponíveis?" → Medicina 24h, Psicologia, Nutrição, Jurídico, Veterinário
7. "E se eu não gostar?" → Garantia de satisfação (definir com cliente)

**Nota:** Os CTAs "Assinar Agora" ficarão como placeholder até a Story 3.2 integrar o Asaas.
Usar `data-plan="cartao"` e `data-plan="pix_boleto"` nos botões para facilitar a integração.

**Referência visual:** saudemental.cuidarhoje.com.br
- Dark hero com preço em destaque
- Nav com apenas 2 itens + CTA
- Seções diretas sem ruído
